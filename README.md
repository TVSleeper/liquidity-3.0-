# NES/USDT PancakeSwap Infinity Liquidity Console

Локальное приложение для управления концентрированной ликвидностью PancakeSwap Infinity v4 на BNB Smart Chain.

## Запуск

```bash
npm install
npm run compile:executor
npm run dev
```

Откройте адрес, который покажет Vite, обычно `http://127.0.0.1:5173`.

## Что умеет

- Подключение MetaMask в BNB Smart Chain.
- Восстановление `PoolKey` по `poolId` через `Initialize`-лог `CLPoolManager`.
- Добавление ликвидности в минимальном допустимом tick-диапазоне.
- Добавление двумя токенами или одним токеном через диапазон выше/ниже текущей цены.
- Частичное и полное снятие ликвидности.
- Развертывание персонального `AtomicLiquidityExecutor`.
- Атомарный сценарий `снять ликвидность -> продать выбранный токен -> вернуть остатки`.
- Монитор swap-событий и подготовка атомарного rebalance-транзакта.

## Найденные параметры NES/USDT pool

- `poolId`: `0x1c0195a12979e395d956a9e2581ef720b925b2a7cc1c60a5b82d9fc0fc564ffa`
- `currency0 / NES`: `0x3131f6B80C26936aB03F7d9D29Eb4Ddf36AC3FB5`
- `currency1 / USDT`: `0x55d398326f99059fF775485246999027B3197955`
- `hooks`: `0xb0BAa371b899950B4Ef6A27c21bAf5ef7c434d0f`
- `fee`: `67`
- `parameters`: `0x00000000000000000000000000000000000000000000000000000000000a0045`
- `tickSpacing`: `10`

Один обычный tick — это примерно `0.01%`, но этот pool разрешает usable ticks с шагом `10`, поэтому минимальный фактический диапазон позиции в нём около `0.100045%`.

## Важное ограничение

Локальный интерфейс с MetaMask не может гарантированно выполнить действие в том же блоке сразу после чужого swap: MetaMask требует ручную подпись, а порядок транзакций контролирует сеть/валидаторы. Для настоящего автоматического same-block исполнения нужен отдельный signer-бот с приватным ключом, private orderflow/bundle-инфраструктура или hook, подключенный к пулу при его создании. Для уже существующего пула hook нельзя добавить задним числом.

`AtomicLiquidityExecutor` делает действия атомарными внутри вашей собственной транзакции: если какая-то часть не проходит, откатывается всё.

## Безопасность

- Не используйте `min out = 0` для реальных сумм.
- Проверяйте адреса `Permit2`, `CLPositionManager`, `UniversalRouter` и `CLPoolManager` перед большими транзакциями.
- Сначала тестируйте маленькой суммой.
- Helper-контракт не хранит средства после успешной операции: он sweeping-ит оба токена обратно владельцу позиции, но всё равно требует NFT approve для конкретной позиции.

## Если пул не загружается

Публичные BNB RPC часто ограничивают диапазон `eth_getLogs`. Приложение читает логи чанками, но стартовый блок всё равно влияет на скорость. Если `PoolKey` не найден, уменьшите `Scan from block`; если поиск идёт слишком долго, увеличьте его ближе к блоку создания пула.
